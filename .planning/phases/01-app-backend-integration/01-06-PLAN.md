    Create AI Chat and Analytics models, repositories, and ViewModels.

    AI Chat Models:
    ```kotlin
    @Serializable
    data class ChatMessage(
        val id: String,
        val role: String, // user, assistant
        val content: String,
        val createdAt: String
    )

    @Serializable
    data class ChatConversation(
        val id: String,
        val title: String,
        val updatedAt: String,
        val messages: List<ChatMessage> = emptyList()
    )

    @Serializable
    data class ChatRequest(
        val message: String,
        val conversationId: String? = null
    )

    @Serializable
    data class ChatResponse(
        val conversationId: String,
        val reply: String
    )
    ```

    AiChatRepository:
    - Methods:
      - suspend fun sendMessage(message: String, conversationId: String? = null): Result&lt;ChatResponse&gt; → POST /ai/chat with timeout 60s
      - suspend fun getConversations(): Result&lt;ConversationsGrouped&gt; → GET /ai/conversations
      - suspend fun getConversation(conversationId: String): Result&lt;ChatConversation&gt; → GET /ai/conversations/{conversationId}
      - suspend fun deleteConversation(conversationId: String): Result&lt;Unit&gt; → DELETE /ai/conversations/{conversationId}
    - The sendMessage endpoint may take up to 60 seconds (AI generation); ensure Ktor client timeout is configured accordingly

    AiChatViewModel:
    - State: AiChatUiState with messages: List&lt;ChatMessage&gt;, conversations: List&lt;ChatConversation&gt;, currentConversationId: String?, inputText: String, isLoading, error, isTyping (AI is generating)
    - Methods: loadConversations(), selectConversation(id), sendMessage(text), deleteConversation(id), startNewChat()
    - Automatically appends user message to list, shows loading, then appends AI response
    - Handles streaming if backend supports SSE (optional; if not, poll for response)

    Analytics Models:
    ```kotlin
    @Serializable
    data class PerformanceAnalytics(
        val overallScore: Float,
        val totalTests: Int,
        val totalQuestions: Int,
        val accuracy: Float,
        val averageTimePerQuestion: Float,
        val strongSubjects: List&lt;SubjectPerformance&gt;,
        val weakSubjects: List&lt;SubjectPerformance&gt;,
        val trend: List&lt;DailyPerformance&gt; = emptyList()
    )

    @Serializable
    data class SubjectPerformance(
        val subject: String,
        val accuracy: Float,
        val questionsAttempted: Int,
        val averageScore: Float
    )

    @Serializable
    data class DailyPerformance(
        val date: String,
        val score: Float,
        val accuracy: Float
    )

    @Serializable
    data class TestAnalytics(
        val testId: String,
        val testName: String,
        val score: Int,
        val rank: Int?,
        val percentile: Float?,
        val accuracy: Float,
        val timeTaken: Int,
        val subjectBreakdown: List&lt;SubjectPerformance&gt;
    )

    @Serializable
    data class LeaderboardEntry(
        val rank: Int,
        val userName: String,
        val avatarUrl: String? = null,
        val score: Int,
        val isCurrentUser: Boolean = false
    )
    ```

    AnalyticsRepository:
    - Methods:
      - suspend fun getPerformance(): Result&lt;PerformanceAnalytics&gt; → GET /user/performance

      - suspend fun getActivity(limit: Int = 50): Result&lt;List&lt;UserActivity&gt;&gt; → GET /user/activity?limit={limit}
      - suspend fun getLeaderboard(testId: String? = null): Result&lt;List&lt;LeaderboardEntry&gt;&gt; → if endpoint exists

    - Methods: loadPerformance(), loadTestAnalytics(), loadActivity(), loadLeaderboard(testId)

    ```kotlin
    @Serializable
    data class SubscriptionPlan(
        val id: String,
        val name: String,
        val description: String? = null,
        val price: Int, // in rupees/paise

        val features: List&lt;String&gt; = emptyList(),

    data class SubscriptionStatus(
        val isActive: Boolean,

    )

    @Serializable
    data class Order(
        val id: String,
        val planId: String,
        val amount: Int,
        val currency: String = "INR",
        val status: String, // created, paid, failed
        val createdAt: String
    )
    ```

    BillingRepository:
    - Methods:
      - suspend fun getPlans(): Result&lt;List&lt;SubscriptionPlan&gt;&gt; → GET /pricing or /billing/plans

    BillingViewModel:
    - State: BillingUiState with plans: List&lt;SubscriptionPlan&gt;, status: SubscriptionStatus?, selectedPlan: SubscriptionPlan?, isLoading, error, order: Order?
    - Methods: loadPlans(), loadStatus(), selectPlan(plan), subscribe(), verifyPayment(paymentId, signature)
    - For this phase, implement plan browsing and selection; payment flow can be documented as TODO if SDK integration is complex

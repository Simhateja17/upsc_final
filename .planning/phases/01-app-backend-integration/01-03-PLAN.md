<objective>
Connect daily practice screens (Daily MCQ, Daily Answer Writing, Editorials/Current Affairs) to backend APIs, replacing sample data with real daily content.
</objective>

<execution_context>
Depends on Plan 01 (ApiClient, ServiceLocator) and Plan 02 (Dashboard patterns). Maps to backend `/daily-mcq/*`, `/daily-answer/*`, `/editorials/*` routes.
</execution_context>

<context>
Backend routes:
- `/daily-mcq/today` → today's MCQ set metadata
- `/daily-mcq/today/questions` → list of MCQQuestion
- `/daily-mcq/today/submit` → POST answers + timeTaken
- `/daily-mcq/today/results` → user's results for today
- `/daily-mcq/today/review` → review with explanations
- `/daily-mcq/today/recommendations` → recommended topics
- `/daily-mcq/practice` → practice questions
- `/daily-answer/today` → today's mains question metadata
- `/daily-answer/today/question` → full question
- `/daily-answer/today/submit-text` → POST answer text
- `/daily-answer/today/upload` → POST file upload
- `/daily-answer/today/evaluation-status` → check evaluation status
- `/daily-answer/today/results` → evaluation results
- `/editorials/today` → today's editorials
- `/editorials/today?source={source}&date={date}` → filtered editorials

API response envelope: `{ status, data, message }`
</context>

    **Models** (`@Serializable` with `ignoreUnknownKeys = true`):

    ```kotlin
    data class DailyMcq(
        val id: String,
        val date: String,
        val title: String,
        val topic: String,
        val tags: List<String> = emptyList(),
        val questionCount: Int = 0,
        val timeLimit: Int = 0, // minutes
        val totalMarks: Int = 0
    )

    data class MCQQuestion(
        val id: String,
        val questionNum: Int,
        val questionText: String,
        val category: String,
        val difficulty: String,
        val options: List<MCQOption>,
        val correctOption: String? = null, // only present in review
        val explanation: String? = null // only present in review
    )

    data class MCQOption(
        val id: String,
        val text: String
    )

    data class MCQAnswer(
        val questionId: String,
        val selectedOption: String?,
        val timeTaken: Int = 0 // seconds
    )

    data class MCQAttempt(
        val id: String? = null,
        val score: Int = 0,
        val totalMarks: Int = 0,
        val correctCount: Int = 0,
        val wrongCount: Int = 0,
        val skippedCount: Int = 0,
        val accuracy: Float = 0f,
        val timeTaken: Int = 0,
        val rank: Int? = null,
        val percentile: Float? = null,
        val strongTopics: List<String> = emptyList(),
        val weakTopics: List<String> = emptyList()
    )

    data class MCQReview(
        val questions: List<MCQQuestion>,
        val attempt: MCQAttempt
    )
    ```

      - `suspend fun getReview(): Result<MCQReview>` → GET `/daily-mcq/today/review`

    **Daily Answer Writing Models:**
    ```kotlin
    @Serializable
    data class DailyMainsQuestion(
        val id: String,
        val date: String,
        val title: String,
        val questionText: String,
        val paper: String,
        val subject: String,
        val marks: Int,
        val wordLimit: Int,
        val timeLimit: Int, // minutes

    )

    @Serializable
    data class MainsAttempt(
        val id: String? = null,
        val answerText: String? = null,
        val fileUrl: String? = null,
        val wordCount: Int = 0,
        val status: String = "pending", // pending, submitted, evaluating, evaluated
        val score: Int? = null,
        val feedback: String? = null,
        val submittedAt: String? = null
    )
    ```

    **DailyAnswerRepository:**
    - Methods:
      - `suspend fun getToday(): Result<DailyMainsQuestion>` → GET `/daily-answer/today`
      - `suspend fun getFullQuestion(): Result<DailyMainsQuestion>` → GET `/daily-answer/today/question`
      - `suspend fun submitText(answerText: String): Result<MainsAttempt>` → POST `/daily-answer/today/submit-text`
      - `suspend fun uploadFile(fileBytes: ByteArray, fileName: String): Result<MainsAttempt>` → POST `/daily-answer/today/upload` (multipart/form-data)
      - `suspend fun getEvaluationStatus(attemptId: String?): Result<MainsAttempt>` → GET `/daily-answer/today/evaluation-status`
      - `suspend fun getResults(attemptId: String?): Result<MainsAttempt>` → GET `/daily-answer/today/results`
    - Note: File upload requires multipart Ktor configuration. Add `ktor-client-content-negotiation` and configure `HttpClient` to support multipart in ApiClient or create a separate upload client.

    **DailyAnswerViewModel:**
    - State: `DailyAnswerUiState` with `question: DailyMainsQuestion?`, `answerText: String`, `attempt: MainsAttempt?`, `isLoading`, `error`, `isSubmitting`
    - Methods: `loadQuestion()`, `updateAnswer(text)`, `submitText()`, `checkStatus()`, `loadResults()`
    - Auto-polls evaluation status after submission (every 10s for up to 2 minutes)

      - `suspend fun bookmark(editorialId: String): Result<Unit>` → if backend has endpoint; otherwise local only

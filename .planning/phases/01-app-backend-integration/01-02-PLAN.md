<objective>
Connect core user-facing screens (Dashboard, Performance, Profile, Settings) to backend APIs, replacing all hardcoded data with real user data.
</objective>

<execution_context>
Depends on Plan 01 (ApiClient, AuthRepository, ServiceLocator). Uses existing backend routes under `/user/*`.
</execution_context>

<context>
Backend routes to use:
- GET `/user/dashboard` → dashboard summary (daysRemaining, trio, todayTasksCount, recentActivity, streak)
- GET `/user/streak` → streak data
- GET `/user/activity?limit={n}` → recent activity
- GET `/user/performance` → performance metrics
- GET `/user/practice-stats` → practice statistics
- GET `/user/test-analytics` → test analytics
- GET `/auth/me` → current user profile
- PUT `/user/profile` → update profile
- PUT `/user/settings` → update settings

ApiResponse envelope: `{ status: "success"|"error", data?: T, message?: String }`
All routes require Bearer token (handled by ApiClient auth interceptor).
</context>

  <action>
    Create domain models and repository for dashboard data.

    **Models** (all `@Serializable` with `ignoreUnknownKeys` support):

    ```kotlin
    data class DashboardData(
        val daysRemaining: Int,
        val trio: DailyTrio,
        val todayTasksCount: Int,
        val recentActivity: List<UserActivity>,
        val streak: UserStreak
    )

    data class DailyTrio(
        val mcq: TrioItem,
        val editorial: TrioItem,
        val mains: TrioItem
    )

    data class TrioItem(
        val status: String, // completed, pending, unavailable
        val topic: String,
        val questionCount: Int? = null
    )

    data class UserActivity(
        val id: String,
        val type: String,
        val title: String,
        val description: String? = null,
        val metadata: JsonObject? = null,
        val createdAt: String
    )

    data class UserStreak(
        val currentStreak: Int = 0,
        val longestStreak: Int = 0,
        val weekActivity: List<Boolean> = emptyList(),
        val lastActiveDate: String? = null
    )
    ```

    **DashboardRepository:**
    - Depends on ApiClient
    - Methods:
      - `suspend fun getDashboard(): Result<DashboardData>` → GET `/user/dashboard`, parse `ApiResponse<DashboardData>`
      - `suspend fun getStreak(): Result<UserStreak>` → GET `/user/streak`
      - `suspend fun getActivity(limit: Int = 10): Result<List<UserActivity>>` → GET `/user/activity?limit={limit}`
      - `suspend fun getPerformance(): Result<JsonObject>` → GET `/user/performance` (generic for now)
      - `suspend fun getTestAnalytics(): Result<JsonObject>` → GET `/user/test-analytics`
    - All methods extract `.data` from ApiResponse, handle errors via Result

    **DashboardViewModel:**
    - State: `DashboardUiState` with `isLoading: Boolean`, `dashboard: DashboardData?`, `error: String?`, `streak: UserStreak?`, `activities: List<UserActivity>`
    - Methods: `loadDashboard()`, `loadStreak()`, `loadActivity(limit)`
    - Automatically calls `loadDashboard()` on init
    - Uses `viewModelScope.launch` for coroutines
  </action>

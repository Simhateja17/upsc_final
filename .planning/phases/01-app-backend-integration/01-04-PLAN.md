<objective>
Connect study tool screens (Mock Tests, PYQ, Flashcards, Mindmaps) to backend APIs, replacing static content with real test data and study materials.
</objective>

<execution_context>
Depends on Plan 01 (ApiClient, ServiceLocator) and Plan 02 (patterns). Maps to backend `/mock-tests/*`, `/pyq/*`, `/flashcards/*`, `/mindmaps/*` routes.
</execution_context>

<context>
Backend routes:
- `/mock-tests` → list mock tests
- `/mock-tests/:id` → get test details
- `/mock-tests/:id/start` → start attempt
- `/mock-tests/:id/questions` → get questions
- `/mock-tests/:id/submit` → submit attempt
- `/mock-tests/:id/results` → get results
- `/pyq` → list PYQ categories
- `/pyq/:year/:paper` → get questions
- `/flashcards` → list decks/topics
- `/flashcards/:topic` → get flashcards
- `/mindmaps` → list mindmaps
- `/mindmaps/:id` → get mindmap details
</context>

    **MockTest Models:**
    ```kotlin
    @Serializable
    data class MockTest(
        val id: String,
        val title: String,
        val description: String? = null,
        val subject: String,
        val duration: Int, // minutes
        val totalMarks: Int,
        val questionCount: Int,
        val difficulty: String,
        val isPremium: Boolean = false,
        val attemptsAllowed: Int? = null,
        val tags: List<String> = emptyList()
    )

    @Serializable
    data class MockTestQuestion(
        val id: String,
        val questionNum: Int,
        val questionText: String,
        val options: List<MCQOption>,
        val marks: Int = 1,
        val negativeMarks: Float = 0f
    )

    @Serializable
    data class MockTestAttempt(
        val id: String? = null,
        val mockTestId: String,
        val score: Int = 0,
        val totalMarks: Int = 0,
        val correctCount: Int = 0,
        val wrongCount: Int = 0,
        val skippedCount: Int = 0,
        val accuracy: Float = 0f,
        val timeTaken: Int = 0,
        val rank: Int? = null,
        val percentile: Float? = null,
        val completedAt: String? = null
    )
    ```

    **MockTestRepository:**
    - Methods:
      - `suspend fun getTests(): Result<List<MockTest>>` → GET `/mock-tests`
      - `suspend fun getTest(id: String): Result<MockTest>` → GET `/mock-tests/{id}`
      - `suspend fun startTest(id: String): Result<MockTestAttempt>` → POST `/mock-tests/{id}/start`
      - `suspend fun getQuestions(testId: String): Result<List<MockTestQuestion>>` → GET `/mock-tests/{testId}/questions`
      - `suspend fun submitTest(testId: String, answers: List<MCQAnswer>, timeTaken: Int): Result<MockTestAttempt>` → POST `/mock-tests/{testId}/submit`
      - `suspend fun getResults(testId: String): Result<MockTestAttempt>` → GET `/mock-tests/{testId}/results`
      - `suspend fun getLeaderboard(testId: String): Result<List<LeaderboardEntry>>` → if endpoint exists

    **MockTestViewModel:**
    - State: `MockTestUiState` with `tests: List<MockTest>`, `selectedTest: MockTest?`, `questions: List<MockTestQuestion>`, `currentQuestionIndex`, `answers`, `attempt: MockTestAttempt?`, `isLoading`, `error`, `screen` (LIST, INSTRUCTIONS, QUIZ, RESULTS)
    - Methods: `loadTests()`, `selectTest(test)`, `startTest()`, `selectAnswer(qId, optId)`, `nextQuestion()`, `previousQuestion()`, `submitTest()`, `loadResults()`
    - Timer tracking for quiz duration
